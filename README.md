# Wing Services List — Gutenberg Block Plugin

A WordPress plugin that registers a custom **Wing Services List** Gutenberg block, allowing content editors to manage and display a list of services directly in the block editor.

---

## Features

| Feature | Detail |
|---|---|
| **Service cards** | Each card holds a title, description, image, optional link URL + label |
| **Inline editing** | Click any text in the editor to edit it in-place (RichText) |
| **Media picker** | Click the image area to open the WordPress media library |
| **Column control** | 1–4 columns, responsive grid (stacks to 1 col on mobile) |
| **Card styles** | Shadow · Border · Flat — selectable in the sidebar |
| **Accent colour** | Colour picker in the sidebar; applied to links, hover states, and the border-style stripe |
| **Heading block** | Optional heading + sub-heading area above the grid, toggleable |
| **Server-side render** | Uses a `render.php` template — no stale serialised markup |
| **Scroll animation** | Cards fade-in on scroll via IntersectionObserver (respects `prefers-reduced-motion`) |
| **RTL support** | Webpack builds an `-rtl.css` variant automatically |

## Installation

### From source (development)

```bash
# 1. Clone the repo into your WordPress plugins directory
cd wp-content/plugins
git clone https://github.com/your-username/wing-services-list

# 2. Install JS dependencies and build assets
cd wing-services-list
npm install
npm run build

# 3. Activate the plugin in WP Admin → Plugins
```

### From a ZIP

1. Download the repository as a ZIP
2. In WP Admin → Plugins → Add New → Upload Plugin
3. Activate

> **Requirements:** WordPress ≥ 6.0, PHP ≥ 8.0, Node 18+ (build only)

## Usage

1. Open any post or page in the block editor
2. Search for **"Wing Services List"** in the block inserter (or type `/wing`)
3. Click **Add Service** to create your first card
4. Click the image area on any card to choose a photo from the media library
5. Click **Edit** on a card to reveal the link URL / label fields
6. Use the **sidebar panel** to adjust columns, card style, accent colour, and heading

## Design Decisions

- **Dynamic block (server-side render)** — chosen over a static save function so the markup can be updated without requiring block validation or content migration. The `render.php` template always uses the latest HTML structure and escaping logic.
- **Gutenberg over Elementor** — the brief allowed either; Gutenberg is the native WordPress approach, ships with zero extra dependencies, and is the long-term direction of the platform.
- **No external libraries** — the editor UI relies entirely on `@wordpress/components` and `@wordpress/block-editor`, matching the native WordPress look and feel.
- **CSS custom properties** — the accent colour chosen in the sidebar is passed as a `--wing-accent` CSS variable, enabling consistent theming with minimal runtime overhead.
- **`@wordpress/scripts`** — provides Webpack, Babel, PostCSS, and Stylelint with sensible defaults; reduces tooling boilerplate significantly.

## What's Outstanding / Known Gaps

Given the 2-hour scope, the following were deprioritised:

| Item | Notes |
|---|---|
| **Drag-to-reorder** | Cards can be reordered via Up/Down arrow buttons. Full drag-and-drop (e.g. `@dnd-kit`) would be a nice upgrade. |
| **Icon picker** | A service icon/badge option (e.g. from Dashicons or an SVG library) could replace or complement the image. |
| **Animation in editor** | The scroll-fade animation only runs on the front-end (`view.js`). |
| **E2E / unit tests** | No automated tests written; would use `@wordpress/jest-console` + Playwright for a production plugin. |
| **Elementor widget** | Only the Gutenberg block was implemented. An Elementor widget would require a separate class extending `\Elementor\Widget_Base`. |
| **REST endpoint caching** | The block is dynamic; adding `get_transient`/`set_transient` in `render.php` for high-traffic sites would be worthwhile. |
